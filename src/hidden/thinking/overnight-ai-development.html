Overnight AI Development

Wake up to implementations that your AI dev team built overnight on trickle power. TDD kanban applied to AI-assisted development.

The Core Insight: Tests ARE Requirements

Traditional broken flow:
"Build a user login system" → Developer guesses → Code doesn't match expectations

AI-assisted TDD flow:
Backlog item → Analyst writes tests → Human PO approves → Dev builds to pass tests

Key breakthrough: The analyst/test-writer does the expensive thinking upfront, creating crystal-clear specifications that eliminate ambiguity.

The Kanban Flow

Backlog → Analysis/Test-Writing → Human PO Approval → Development → Validation
           (cheap compute)            (human decision)    (expensive compute)

Stage Breakdown

  1. Backlog: Human adds high-level feature requests
  2. Analysis: AI analyst breaks down into testable requirements
  3. PO Approval: Human reviews/approves/rejects the test specifications
  4. Development: AI dev agent implements to pass the approved tests
  5. Validation: Automated testing confirms requirements met

File-Based Kanban Implementation 

project/
├── backlog/
│   ├── feature_login.md
│   └── enhancement_ui.md
├── analysis/
│   └── login_tests_and_requirements.md
├── approved/
│   └── login_approved.json
├── development/
│   ├── login_implementation.py
│   └── login_tests.py
├── testing/
│   └── test_results/
└── done/
    └── completed_features/

Everything is visible, debuggable, and version-controlled. No magic databases, no hidden state. Just files moving through directories.

The Overnight Vision

"Wake up to implementations that your AI dev team built overnight on trickle power. Because code doesn't sleep, but developers do."

The Workflow

  1. Before bed: Review and approve analyses
  2. Overnight: Start development agent (low priority, trickle power)
  3. Morning: Check what got built with the morning report

Power-Efficient Orchestration

  • Resource-aware processing: Check system load before expensive operations
  • Adaptive sleep: Slow down when system is busy
  • Model swapping: Only load one model at a time on low-memory systems
  • Rate limiting: Built-in delays for trickle-power operation

Safety and Rollback Strategy

The Overnight Safety Net:

  • Before starting: Git commit checkpoint with stable tag
  • After development: Commit overnight work
  • If something went wrong: Reset to stable tag

Incremental Validation:

  • Each implementation includes its own tests
  • Manual review before moving to production
  • Rollback capability at every stage
  • Version control for all artifacts

Ultimate Vision: Autonomous Dev Team

The End Goal:

A self-managing development team that:

  • Processes your backlog while you sleep
  • Produces tested, documented implementations
  • Operates on minimal power (raspberry pi + solar panel)
  • Never bricks your system (everything is reversible)
  • Wakes you up with working code and clear status reports

You remain the Product Owner, making strategic decisions about what to build. The AI handles the tactical work of how to build it, following your approved specifications exactly.

Pure steampunk engineering: Visible processes, replaceable components, human oversight, and the beautiful simplicity of files and scripts doing real work.

The Morning Workflow

  1. Check the morning report: See what got built overnight
  2. Review implementations: Browse the development directory
  3. Run integration tests: Verify new code works with existing system
  4. Approve good implementations: Move to done
  5. Fix issues: Add clarifications or send back to analysis
  6. Plan next night: Add new items to backlog
